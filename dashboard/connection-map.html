DATA FLOW (SHOULD BE)
1. USER → Browser → localhost:30889 → Gateway Pod

2. Gateway → Redis Pub/Sub:
   - Publishes message to channel "crod.requests"
   - Message format: {"type": "analyze", "data": "...", "id": "uuid"}

3. Districts subscribe to Redis:
   - Meta-Chain listens on "crod.requests"
   - Decides which district handles what
   - Publishes to specific channels:
     * "crod.pattern" → Pattern District
     * "crod.memory" → Memory Quarter
     * "crod.intelligence" → Intelligence Hub

4. Districts process and respond:
   - Publish results to "crod.responses.{id}"
   - Gateway listens for responses
   - Returns to user via HTTP/WebSocket
