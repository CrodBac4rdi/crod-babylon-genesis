# 🗂️ GitHub Projects & Doku-Guide für CROD

## Was ist GitHub Projects?
GitHub Projects ist ein mächtiges Kanban-Board/Projektmanagement-Tool direkt in GitHub. Du kannst damit:
- Aufgaben (Issues, Pull Requests, freie Tasks) planen, priorisieren und tracken
- Roadmaps, Backlogs, Sprints und Status visualisieren
- Automatisierungen (z.B. ToDo → In Progress → Done) nutzen
- Doku, ToDos, Bugs, Features, Ideen und Fortschritt zentral verwalten

## Wie nutze ich GitHub Projects für CROD?
1. **Gehe auf dein Repository auf GitHub.**
2. Klicke oben auf den Tab "Projects" (oder "Projekte").
3. Erstelle ein neues Project (z.B. "CROD 2025 Roadmap").
4. Wähle "Board" (Kanban) oder "Table" (Tabellenansicht).
5. Lege Spalten an: z.B. "Ideen", "ToDo", "In Progress", "Review", "Done".
6. Füge Issues, Pull Requests oder freie Tasks als Karten hinzu.
7. Ziehe Karten per Drag&Drop durch die Spalten.
8. Nutze Filter, Automatisierungen und Status für Übersicht.

## Best Practices für CROD
- **Jede größere Aufgabe als Issue anlegen** (z.B. "Quantum Mining für Pattern District")
- **Labels nutzen**: z.B. `bug`, `feature`, `enhancement`, `security`, `doc`, `urgent`
- **Verlinke Issues in Pull Requests** (`closes #123`)
- **Nutze die Roadmap-Ansicht für langfristige Planung**
- **Verknüpfe Doku/ToDos aus den Modul-Dokus direkt als Issues/Tasks**

## Beispiel-Workflow
1. In `docs/MODUL-QUANTUM.md` steht ein ToDo: "Quantum-API für externe Tools".
2. Lege dazu ein Issue an: "Quantum-API für externe Tools implementieren".
3. Ziehe das Issue ins Project-Board unter "ToDo".
4. Wenn du daran arbeitest, schiebe es auf "In Progress".
5. Nach Merge auf "Done".
